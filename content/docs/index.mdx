Glyphfield is a local-first brand studio for defining one identity and applying it across product, marketing, developer, motion, and physical design.

The Studio and agent API share the same built-in identities, catalog, and generation language. Humans can work visually in the browser; agents can discover structured capabilities and generate portable artifacts over HTTP.

## The system

Glyphfield is organized around four layers:

1. **Identity foundations** — name, positioning, voice, colors, typography, and logo family.
2. **Reusable generators** — backgrounds, logos, slides, OpenGraph images, terminal cards, components, and motion.
3. **Brand applications** — 47 digital, product, developer, social, editorial, event, and physical elements.
4. **Portable outputs** — PNG, SVG, GIF, and JSON artifacts that can leave the Studio.

## Local-first by default

Browser projects and uploaded files remain in the browser. The public generation API processes each request in memory and does not persist the submitted identity or asset. Remote asset URLs are intentionally unsupported.

Brand settings is the source of truth for the active project. Edits persist immediately and flow into all tools that read the identity.

## Core identity

Define the brand name, short name, website, tagline, positioning, and description. The short name is used for monograms and compact marks when a logo is unavailable.

## Logo family

Upload separate marks for light and dark surfaces. Use SVG or a transparent PNG under 1.5 MB.

- The **dark mark** is intended for light surfaces.
- The **light mark** is intended for dark surfaces.

Glyphfield chooses the appropriate variant in project tabs, templates, previews, and exports. If a matching variant is missing, it falls back to the available mark.

## Semantic colors

Each color contains a name, role, and six-digit HEX value. The color tools also expose an OKLCH representation for perceptual comparison.

The standard roles are ink, paper, emphasis, and success. Generators use ink and paper as default foreground and background values.

## Typography

Assign families to Display, Body, Accent, and Code roles. The identity-wide family names feed templates, components, and the brand board. Individual tools may also load a local font for a one-off artifact.

## Language and voice

Greetings feed the default animation sequence. Voice principles and preferred phrases seed titles and supporting copy across templates. Audiences document who the system should address.

## What updates automatically

- Moodboard foundations and applications
- Brand element defaults
- OpenGraph, slides, blog, and partnership templates
- Logo, color, typography, and component previews
- Animation greetings and default brand colors
- Agent element briefs after the identity is supplied in a request

Existing per-tool overrides remain overrides. Reset a tool or element to return to its identity-derived defaults.
